# Deckgame V0.1 — Notes de version

**Date :** 2026-06-08
**Tag :** `v0.1.0`
**Branche :** master

---

## Résumé

V0.1 est la première release mobile/PWA de Deckgame.
Elle consolide V0.0.0 (moteur stable, 52 cartes) avec :
- Support mobile complet (portrait 390×844, paysage jusqu'à 740×340)
- PWA installable (manifest, standalone, fullscreen)
- Accessibilité touch améliorée (touch targets ≥ 44px)
- Documentation utilisateur complète

---

## Nouveautés V0.1 (patchs 0045–0049)

### PWA & Fullscreen (PATCH 0045)
- `manifest.webmanifest` : display standalone, orientation any
- `useFullscreen` hook : API Fullscreen standard + WebKit (iOS Safari)
- `height: 100dvh` (dynamic viewport height)
- Metas PWA dans `index.html` (`viewport-fit=cover`, `apple-mobile-web-app-*`)
- Bouton "Plein écran" dans StartScreen et plateau de jeu

### Audit mobile multi-viewport (PATCH 0046)
- Tests Playwright 25/25 sur 4 viewports (portrait + 3 paysage)
- Classification P0/P1/P2/P3 des problèmes identifiés
- Aucun P0 trouvé — application jouable sur tous les viewports

### Corrections mobile (PATCH 0047)
- Journal portrait : bouton × de fermeture + header cohérent
- StartScreen : bouton "Plein écran" secondaire (alignSelf center, taille réduite)

### Accessibilité & touch (PATCH 0048)
- Touch targets portrait ≥ 44px : Journal, Fin du tour
- Touch targets landscape ≥ 36px : Journal, Fin du tour, Attaquer
- `aria-label="Fermer le journal"` sur le bouton × landscape
- Abandonner portrait : 22px → 32px

### Documentation (PATCH 0049)
- README mis à jour avec stack, guides, historique
- `docs/user-guide-v0.1.md` : règles, interface, boutons
- `docs/mobile-pwa-guide.md` : PWA, orientations, limitations

## Héritage V0.0.0

- 52 cartes Star Realms set de base (toutes factions + effets alliés/scrap)
- Moteur immuable TypeScript, 0 mutation d'état
- Bot déterministe, 4 profils de jeu
- Mode solo + 2 joueurs local
- Simulation batch 100 parties (taux victoire ~50% ±5%)
